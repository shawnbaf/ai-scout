### 1. Claude Code v2.1.90 — Hook Reliability Fixes

CC v2.1.90 fixes two bugs that directly affect the founder's hook setup:

- **PostToolUse format-on-save:** Edit/Write was failing with "File content has changed" when a PostToolUse hook (like auto-format) rewrote the file between consecutive edits. Now fixed — auto-format hook works reliably across rapid multi-edit sequences.
- **PreToolUse JSON stdout:** Hooks emitting JSON to stdout were not correctly blocking tool calls. Now fixed — git-guard hook using JSON output for structured blocking now works as intended.
- **Auto mode boundaries:** Fixed auto mode not respecting explicit user boundaries. Scheduled tasks in auto mode now correctly honor permission rules.

**Cross-system impact:**
- **Hooks:** Both git-guard (PreToolUse) and auto-format (PostToolUse) had silent failures that are now fixed. Update immediately.
- **Ops Pipeline (Phase 8):** Scheduled tasks in auto mode were ignoring explicit boundaries — critical fix for autonomous operation safety.
- **CLAUDE.md:** No doc changes needed, but verify hooks work correctly after upgrading.

**Action:** Update CC to v2.1.90. Test git-guard and auto-format hooks to confirm fixes.

### 2. Playwright v1.59.0 Breaking API Changes + v1.59.1 Hotfix

New details on yesterday's Playwright v1.59.0 report — specific breaking changes identified:

- **`snapshotForAI()` merged into `ariaSnapshot()`** — any UI review route crawler code calling `snapshotForAI()` will break.
- **`Browser.launchOptions` and `Browser.userDataDir` removed** — scripts accessing these properties after launch will throw.
- **`isClosed` replaced `isClosedOrClosing`** — rename required if referenced.
- **Screencast API migrated from events to `onFrame` callback** — recording scripts using event-based screencast will need rewriting.
- **v1.59.1 hotfix** (also April 1): Reverted a change that broke codegen, `--ui`, and `show` commands. Install v1.59.1, not v1.59.0.

**Cross-system impact:**
- **UI Review (Track 5):** Route crawler must be checked for `snapshotForAI()` and `Browser.launchOptions` usage before upgrading.
- **Demo Video (Phase 10):** Recording pipeline scripts need migration if using event-based screencast.
- **Demo Navigator (Phase 9):** Any Playwright-driven interaction automation needs compatibility check.

**Action:** Pin Playwright at current version until route crawler and demo recording scripts are verified against v1.59.x breaking changes. When ready, upgrade directly to v1.59.1 (skip v1.59.0).

Source: github.com/microsoft/playwright/releases (v1.59.0, v1.59.1)

## Opportunities

### 3. CC v2.1.90 — Performance Fixes for Long Sessions

Three performance fixes that directly improve long-running session quality:

- **SSE transport:** Large streamed frames now processed in linear time (was quadratic). Affects any session with large tool outputs.
- **SDK transcript writes:** Long conversations no longer slow down quadratically. Affects extended dev sessions and scheduled tasks.
- **MCP schema lookup:** Eliminated per-turn `JSON.stringify` of MCP tool schemas on cache-key lookup. Reduces overhead for every turn in sessions with MCP servers connected.

**Cross-system impact:**
- **This scout:** Long web fetch outputs were contributing to session slowdown. Linear SSE processing directly helps.
- **Ops Pipeline (Phase 8):** Scheduled tasks that run for extended periods benefit from all three fixes.

Source: github.com/anthropics/claude-code/blob/main/CHANGELOG.md (v2.1.90)

### 4. CC v2.1.90 — Resume Cache Fix

Fixed `--resume` causing a full prompt-cache miss on the first request for sessions with deferred tools, MCP servers, or custom agents. Previously, resuming a session with MCP connections would re-process the entire conversation context, adding latency and cost.

**Why it matters:** Any workflow that pauses and resumes sessions (the defer pattern from v2.1.89, MCP-connected scheduled tasks) was paying a hidden performance tax on resume. Now fixed.

Source: github.com/anthropics/claude-code/blob/main/CHANGELOG.md (v2.1.90)

### 7. AI Video Production Paradigm — Ingredients-to-Video

Professional AI video workflows in 2026 have shifted from "describe a video" (text-to-video) to "provide key visuals + animate" (ingredients-to-video). Quality is determined by input assets, not generation parameters. Creators average 3.4 tools per workflow. API availability is now the headline feature for developer adoption.

**Why it matters:** The demo video pipeline (Phase 10) currently generates from script + recording. The ingredients model suggests starting from product screenshots and key visual assets, then animating — potentially higher quality with less prompt engineering. Worth evaluating when the pipeline is next updated.

Source: ltx.studio/blog/ai-video-workflow, vo3ai.com/blog/ai-video-workflows
