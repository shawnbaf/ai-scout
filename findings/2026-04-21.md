## Opportunities

### 1. CC Skill Hooks Now Fire in Agent Mode

Claude Code v2.1.116 (April 20) fixes `hooks:` in skill frontmatter so
they fire when CC runs as a main-thread agent via `--agent`. Previously,
skill-defined hooks were silently ignored in agent mode.

**Why it matters:** The founder's architecture uses self-growing skills
(project-health.md, error-triage.md, visual-qa/SKILL.md,
fix-guardrails.md) and runs CC as scheduled tasks. If any skill defines
frontmatter hooks, those hooks now actually execute in agent/scheduled
contexts. This extends the guardrail pattern — the same safety nets that
work in interactive sessions now work in automated runs.

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** Scheduled tasks and agent-mode runs now
  respect skill-level hooks, strengthening autonomous operation safety.
- **Self-Growing Skills:** Skills can now carry their own hooks into any
  execution context, making them more self-contained.
- **CLAUDE.md:** Consider documenting which skills should define
  frontmatter hooks for agent-mode guardrails.

### 2. CC /resume 67% Faster on Large Sessions

The same release makes `/resume` 67% faster on sessions over 40MB, with
improved dead-fork handling. Large sessions are common in complex
projects with extensive tool use history.

**Why it matters:** The founder works on multi-phase projects with long
sessions. Faster resume means less dead time when reconnecting to
interrupted sessions. Dead-fork handling improvement means less wasted
context on abandoned conversation branches.

**Cross-system impact:**
- **Daily CC workflow:** Direct productivity improvement across all
  projects. No action needed — automatic on update.

## Bookmarked

### 3. CC v2.1.116 Maintenance Improvements

Several smaller improvements in the same release:

- **Plugin auto-install:** `/reload-plugins` and background auto-update
  now auto-install missing dependencies. Reduces manual plugin
  maintenance.
- **Faster MCP startup:** Multiple stdio MCP servers start concurrently;
  `resources/templates/list` deferred to reduce startup blocking.
  Relevant when using multiple MCP servers (GitHub, Composio, etc.).
- **Bash rate limiting:** `gh` command rate limit hints help agents back
  off gracefully instead of hammering GitHub API. Useful for agent-driven
  CI/CD workflows.
- **Thinking progress indicators:** Inline progress messages ("still
  thinking", "thinking more", "almost done thinking") provide visibility
  during long reasoning chains. UX polish, not workflow-changing.
