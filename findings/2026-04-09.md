## Action Needed

### 1. Claude Code v2.1.97 — MCP Memory Leak + Subagent Worktree Leak

Two critical fixes in v2.1.97 that directly affect the founder's workflows:

- **MCP HTTP/SSE memory leak:** MCP connections were accumulating ~50 MB/hr of unreleased buffers on server reconnect. Any session with MCP servers (Composio, GitHub MCP, Sentry MCP, etc.) was slowly consuming memory during long-running sessions.
- **Subagent worktree CWD leak:** Subagents with worktree isolation or `cwd:` override were leaking their working directory back to the parent session's Bash tool. This means parallel agents using worktrees could have been polluting each other's file operations.

Additional fixes in v2.1.97 worth noting:
- Permission rules with JavaScript prototype property names (e.g., `toString`) silently caused `settings.json` to be ignored entirely.
- `--dangerously-skip-permissions` was being silently downgraded to accept-edits mode.
- `permissions.additionalDirectories` changes in settings didn't apply mid-session.
- Compaction was writing duplicate multi-MB subagent transcript files on prompt-too-long retries.

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** MCP-connected scheduled tasks and long-running sessions were leaking memory. Upgrade immediately.
- **This scout:** Uses MCP servers — was affected by the memory leak.
- **Multi-agent workflows:** Subagent worktree leak means parallel agent isolation wasn't fully working. Any worktree-based parallel work should be re-verified after upgrade.
- **CLAUDE.md / Hooks:** If any permission rule names collided with JS prototype properties, the entire settings.json was silently ignored — meaning hooks and permission rules may not have been active.

**Action:** Update CC to v2.1.97 immediately. Verify MCP-connected sessions run without memory accumulation. Re-test parallel worktree workflows.

Source: github.com/anthropics/claude-code/blob/main/CHANGELOG.md (v2.1.97, April 8)

### 2. Claude Code v2.1.94 — Default Effort Level Changed to High

Default effort level changed from Medium to High for API-key, Bedrock/Vertex/Foundry, Team, and Enterprise users. This affects how much reasoning CC applies per turn — High uses more tokens for deeper thinking.

**Why it matters:** CC sessions will now consume more tokens by default. For scheduled tasks and long-running sessions, this increases cost. The founder can control this via `/effort` per session or via settings.

**Cross-system impact:**
- **All CC sessions:** More tokens used per turn by default. Quality may improve, but cost increases.
- **Scheduled tasks:** Higher token consumption on autonomous runs unless explicitly set to Medium.
- **CLAUDE.md:** Consider adding effort level preference to session initialization if cost management is a concern.

**Action:** Be aware of increased token usage. If cost is a concern, set effort to Medium via `/effort` in sessions or configure in settings.

Source: github.com/anthropics/claude-code/blob/main/CHANGELOG.md (v2.1.94, April 7)

## Opportunities

### 3. Claude Managed Agents — Public Beta Launch

Anthropic launched Claude Managed Agents in public beta (April 8). This is a fully managed agent harness that runs Claude as an autonomous agent on Anthropic's infrastructure — separate from Claude Code.

**Key details:**
- Pre-built agent harness with sandboxed cloud containers
- Built-in tools: Bash, file operations (read/write/edit/glob/grep), web search/fetch, MCP servers
- SSE streaming for real-time event monitoring
- Stateful sessions with persistent file systems
- `ant` CLI for managing agents from terminal
- Available to all API accounts (no waitlist), requires `managed-agents-2026-04-01` beta header
- Research previews for: outcomes, multi-agent, and memory (requires access request)

**How it compares to CC scheduled tasks:**
- Managed Agents: API-driven, containerized, designed for production workloads. You define agents (model + system prompt + tools + MCP) and environments (packages + network + files) separately, then start sessions against them.
- CC scheduled tasks: Built on the Claude Code CLI, run as cloud tasks, use the CC tool ecosystem.
- Key difference: Managed Agents is infrastructure for building agent-powered products. CC scheduled tasks are for personal automation.

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** Managed Agents could complement scheduled tasks for workloads needing containerized environments or longer execution windows.
- **This scout:** Could potentially run as a Managed Agent with a defined environment and tools.
- **CLAUDE.md:** The `/claude-api` skill was updated in v2.1.97 to cover Managed Agents.

Source: platform.claude.com/docs/en/managed-agents/overview, releasebot.io/updates/anthropic

### 4. CC v2.1.97 — Focus View + Agent Running Indicator

Two UX improvements for session management:

- **Focus view toggle (Ctrl+O):** In NO_FLICKER mode, shows prompt, one-line tool summary with edit diffstats, and final response. Collapses the noise of long sessions to just decisions and outcomes.
- **`● N running` indicator in `/agents`:** Shows live subagent instance count next to each agent type. Instant visibility into how many parallel agents are active.

**Cross-system impact:**
- **Ops Pipeline:** Focus view helps review what a scheduled task or autonomous session actually did. The agent indicator helps monitor parallel agent orchestration.
- **Multi-agent workflows:** Live count eliminates the "how many are still running?" question.

Source: github.com/anthropics/claude-code/blob/main/CHANGELOG.md (v2.1.97, April 8)

### 5. CC v2.1.94 — Session Title via Hooks

**Why it matters:** Auto-naming sessions from hooks eliminates the manual step of titling sessions. The founder could use this to auto-categorize sessions (e.g., prefix with project name, task type) for better organization when reviewing past sessions.

**Cross-system impact:**
- **Ops Pipeline:** Scheduled tasks could auto-title based on the task being run.
- **Session management:** Easier to find and resume sessions when they're properly named.

### 6. Official `frontend-design` Plugin

Anthropic shipped an official `frontend-design` plugin in the Claude Code marketplace. It provides a `/frontend-design` skill that guides CC to create distinctive, production-grade UI — explicitly designed to avoid "AI slop" aesthetics.

**Key approach:**

- Produces production-grade, functional code with clear aesthetic point-of-view

**Cross-system impact:**

## Bookmarked

### 7. CC v2.1.97 — Rate Limit Retry Backoff Fixed

429 retries were burning all retry attempts in ~13 seconds when the server returned a small `Retry-After` header. Exponential backoff now applies as a minimum. This means CC sessions hitting rate limits will recover more gracefully instead of exhausting retries instantly.

Source: github.com/anthropics/claude-code/blob/main/CHANGELOG.md (v2.1.97, April 8)

### 8. Anthropic Restricts Subscriptions to First-Party Tools

Starting April 4, Claude subscription users can no longer use their subscription limits for third-party harnesses including OpenClaw. Anthropic cites cost optimization — first-party tools like Claude Code are built to maximize prompt cache hit rates, while third-party harnesses are not. The restriction will extend to all third-party harnesses.

**Why it matters:** Doesn't affect the founder directly (uses CC natively). But signals Anthropic is tightening the subscription model around first-party tools. Worth watching if future tools the founder evaluates rely on subscription access.

Source: news.ycombinator.com/item?id=47633396, techcrunch.com/2026/04/04/
