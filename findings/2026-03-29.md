- **Build-verification PreToolUse hook** — Community pattern for a `PreToolUse` hook on `Bash` that intercepts `git commit` commands and automatically runs the project build + test suite before allowing the commit through. If the build fails, the hook blocks the commit and feeds the error back to CC as context. The founder's templates already specify "Build must pass before every commit" as a Hard Rule (`project-audit.md` → Git & Deploy) and `new-project-master.md` lists "CC runs build check before every commit." Currently this relies on CC remembering the rule. A hook would enforce it automatically, similar to how `git-guard` enforces branch protection. Implementation: add a `PreToolUse` hook on `Bash` with `if` field filtering to `git commit`, run `npm run build` (or equivalent), exit code 2 with stderr on failure.
