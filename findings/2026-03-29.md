## Opportunities

- **Build-verification PreToolUse hook** — Community pattern for a `PreToolUse` hook on `Bash` that intercepts `git commit` commands and automatically runs the project build + test suite before allowing the commit through. If the build fails, the hook blocks the commit and feeds the error back to CC as context. The founder's templates already specify "Build must pass before every commit" as a Hard Rule (`project-audit.md` → Git & Deploy) and `new-project-master.md` lists "CC runs build check before every commit." Currently this relies on CC remembering the rule. A hook would enforce it automatically, similar to how `git-guard` enforces branch protection. Implementation: add a `PreToolUse` hook on `Bash` with `if` field filtering to `git commit`, run `npm run build` (or equivalent), exit code 2 with stderr on failure.

## Bookmarked

- **MCP file-watcher server** — Reddit r/ClaudeAI discussion mentioned an MCP server that watches local files for changes and notifies CC to take action. Could bridge external tools (e.g., Figma export, design tool sync) into the CC pipeline. The founder's current design-to-code pipeline is Claude.ai JSX artifacts → CC-PROMPT, so this doesn't directly improve the current workflow. However, if the founder ever adds external tool integrations that produce file artifacts, this pattern could automate the handoff. Needs verification — may be early-stage or unmaintained.

- **AI-generated Remotion compositions** — Pattern for using Claude API to dynamically generate Remotion scene composition code from structured JSON scene descriptions. Instead of hand-coding each scene's React component, a script takes `demo-script.json` and generates the Remotion composition code. Could reduce per-project setup cost for the demo video pipeline. The founder's pipeline already works well with manual components read via `inputProps` and `calculateMetadata()`. This "AI generates composition code" approach adds a generation step and potential fragility. Worth monitoring as the pattern matures but not worth adopting now.

- **Screenshot-to-code improvements** — Several tools have improved screenshot-to-code capabilities in 2026, with some supporting full-page screenshots → component-level code extraction from CLI. The founder's visual QA system captures screenshots and flags issues. If screenshot-to-code tools become reliable enough, they could eventually automate part of the UI review flag → fix cycle (screenshot diff → generated fix code → CC applies). Current tools aren't reliable enough for production use. Worth monitoring.
