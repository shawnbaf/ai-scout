## Opportunities

### 3. PostToolUse Hooks Can Now Replace Tool Output for ALL Tools

Previously, `hookSpecificOutput.updatedToolOutput` only worked for MCP tool
calls. Now it works for all tools — Bash, Read, Edit, Write, etc. This means
hooks can transform, filter, or augment the output of any tool call before
Claude sees it.

**Use cases:**
- Strip sensitive data from Read/Bash output before it enters context
- Inject project-specific context into tool results
- Compress verbose CLI output (complementary to RTK approach below)
- Add metadata to file reads (e.g., "this file was last modified by X")

**Cross-system impact:**
- **Hooks architecture:** The `hookSpecificOutput.updatedToolOutput` field
  becomes a general-purpose output transformation layer. Could affect how
  auto-format and doc-drift-reminder hooks work.
- **new-project-master.md:** Track 7 (Hooks) documentation could mention
  this capability for custom hooks.

### 4. RTK (Rust Token Killer) — Hook-Based CLI Output Compression

Single Rust binary, zero dependencies. Installs as a PreToolUse hook that
transparently rewrites Bash commands (e.g., `git status` → `rtk git status`).
Claims 60-90% token reduction on CLI output. Real examples: `cargo test`
155 lines → 3 lines (98%), `git status` 119 chars → 28 chars (76%).

Uses filtering, grouping, truncation, deduplication. 100+ supported commands,
<10ms overhead. Claude never sees the rewrite — the hook intercepts before
the tool call.

**Workflow impact:** Direct cost/context savings for CC sessions. The hook
architecture is correct — transparent, deterministic, no token cost. For a
solo founder watching costs across multiple projects, this compounds.

**Limitation:** Only works on Bash tool calls, not Read/Grep/Glob built-ins.

**Cross-system impact:**
- **Hooks:** Would be a 4th hook alongside git-guard, auto-format,
  doc-drift-reminder. Evaluate whether the savings justify the dependency.
- **Scheduled tasks:** Token savings compound in long-running tasks like
  this scout.

### 5. Next.js 16 Cache Components = Canonical PPR Path

The experimental `ppr` flag is removed. `cacheComponents: true` in
next.config is the new opt-in. The `use cache` directive replaces implicit
caching. Docs updated April 10, 2026. LCP can be sub-100ms from edge with
static shell caching.

This is the Next.js caching model going forward. The project templates
should reference this instead of experimental PPR flags.

**Cross-system impact:**
- **new-project-master.md:** Phase 2 (Architecture) should reference
  Cache Components when proposing Next.js architecture.
- **CLAUDE.md template:** Build/dev commands and caching instructions
  should reference `cacheComponents` + `use cache`.
- **enhancement-standards.md:** Performance patterns section could
  reference static shell caching for sub-100ms LCP.

### 11. HERMES.md String in Commit History Reroutes Billing

If the case-sensitive string `HERMES.md` appears in recent git commit
messages, CC routes API requests to "extra usage" billing instead of Max
plan quota. CC includes recent commits in the system prompt; a server-side
substring filter misclassifies the request. One user lost $200. Anthropic
confirmed and issued refunds by April 30.

Avoid this string in commit messages until a CC patch ships. Not currently
a risk for the founder's projects (no such commit messages), but important
to know for any automated commit generation.

### 12. React CVE-2026-23864 — Server Components DoS (CVSS 7.5)

Denial of service via memory exhaustion in React Server Components. Affects
React 19.0.0-19.2.3. Patched in React 19.0.4, 19.1.5, 19.2.4. Next.js
patched in 15.0.8+, 15.1.12+, 15.2.9+, 15.3.9+, 15.4.11+, 15.5.10+.

Disclosed Dec 2025, updated Jan 2026. If projects are on Next.js 16.x,
they should already be patched. Verify with `npm ls react-server-dom-webpack`.

### 13. React Compiler Real-World Performance Data

Production numbers maturing: Wakelet reports 10% LCP improvement, 15-30%
INP improvement. Instagram sees 3% average improvement across all pages.
Compiler is stable since React 19 (October 2025). Key limitation: optimizes
re-renders, not architecture — won't fix client waterfalls or unvirtualized
lists. Worth enabling in new projects.

### 14. CSS Subgrid Widely Available

layouts, form layouts, dashboard grids.

### 15. Rive AI Agent Free + Luau Scripting
