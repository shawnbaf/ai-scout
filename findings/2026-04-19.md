## Action Needed

### 1. Claude Haiku 3 Retired Today — Check All Projects

The `claude-3-haiku-20240307` model ID stopped accepting requests on
April 19, 2026. There is no automatic migration — API calls using this
model ID now return errors. This was announced February 19, 2026, with
the retirement date set for today.

**Migration target:** Claude Haiku 4.5 (`claude-haiku-4-5-20251001`).

**Cost impact:** Haiku 4.5 is approximately 4x more expensive than
Haiku 3 was. For high-volume, cost-sensitive API calls (validation,
classification, simple routing), this may change the economics.

**Breaking change in Haiku 4.5:** Cannot set both `temperature` AND
`top_p` parameters simultaneously. If any project code sets both,
one must be removed or the request will fail.

**Why ACTION NEEDED:** Hard cutoff with no fallback. Any project
referencing the old model ID is broken now. The founder should grep
all project codebases for `claude-3-haiku` and `haiku-20240307` and
update any matches to `claude-haiku-4-5-20251001`.

**Cross-system impact:**
- **ARCHITECTURE.md templates:** Any project ARCHITECTURE.md that
  specifies Haiku 3 as the model for cost-sensitive operations needs
  updating.
- **API integration code:** Any Anthropic API client initialization
  or request code pinning to the old model ID.
- **Cost projections:** If any project relies on Haiku 3 pricing for
  budget calculations, the 4x increase needs to be factored in.
- **CLAUDE.md model references:** Check the latest model ID guidance
  in project CLAUDE.md files.

Source: platform.claude.com/docs/en/about-claude/model-deprecations,
platform.claude.com/docs/en/release-notes/overview (Feb 19 announcement),
findskill.ai/blog, claudelab.net/en/blog.
