## Action Needed

### 1. Claude Opus 4.7 GA — Model Upgrade with Cost Implications

Anthropic released Claude Opus 4.7 on April 16, available across Claude API,
Bedrock, Vertex AI, and Microsoft Foundry. Same pricing as Opus 4.6
($5/$25 per million tokens), but a new tokenizer may use 1x–1.35x as many
tokens for the same text — effectively up to 35% higher cost per task.

**Performance improvements:**
- SWE-bench Verified: 80.8% → 87.6% (+7 points)
- SWE-bench Pro: 53.4% → 64.3% (+11 points)
- CursorBench: 58% → 70% (+12 points)
- First Claude model with high-resolution image support (max 2576px / 3.75MP)
- Proactive output verification — model writes tests, runs them, fixes
  failures before declaring a task complete in agentic contexts

**New capabilities:**
- **xhigh effort level** between high and max. At 100k tokens, xhigh scores
  71% — already ahead of Opus 4.6's max at 200k tokens. Anthropic recommends
  starting with high or xhigh for coding/agentic use.
- **Task budgets (public beta)** — token ceilings for agentic loops. Model
  sees a running countdown and finishes gracefully as budget is consumed.
  Activate via `task-budgets-2026-03-13` beta header + `output_config.task_budget`.
- **File-system memory improvements** — better at writing and leveraging
  scratchpad/notes files across turns in agentic contexts.

**Why ACTION NEEDED:**
The model powering every CC-driven workflow got a significant upgrade. The
founder needs to:
1. **Check tokenizer cost impact** — run the same CC-PROMPT through 4.6 and
   4.7 and compare token consumption. If the 35% increase holds, scheduled
   tasks (including this scout) will cost more.
2. **Evaluate xhigh effort** — for complex CC-PROMPTs (Phase 6.5 screen
   design, Phase 10 demo video), xhigh may produce better results than high
   while being cheaper than max. Test the tradeoff.
3. **Update CLAUDE.md model references** — if switching to 4.7, update any
   model ID references. Note: CC currently defaults to 4.6 per session.
4. **Leverage high-res vision** — UI review (Track 5) screenshots can now
   be analyzed at much higher fidelity. The visual-qa skill should benefit
   from 3x better image resolution without any code changes.

**Cross-system impact:**
- **UI Review (Track 5):** High-res vision (2576px) means more accurate
  compliance checks and enhancement suggestions on screenshots. Pass 2
  enhancement suggestions should catch finer visual details.
- **Screen Design (Phase 6.5):** Better vision for evaluating design
  screenshots during the lock phase. Better coding for generating
  component code from locked designs.
- **Demo Video (Phase 10):** Better vision for reviewing Remotion
  composition frames. Better coding for generating Remotion code.
- **Health System (Track 4):** Better coding accuracy should reduce bugs
  in generated code, potentially fewer health issues to triage.
- **CLAUDE.md template:** Should reference xhigh effort level as an
  option. Model ID guidance may need updating.
- **Scheduled tasks:** Monitor token consumption after upgrade. Task
  budgets (when GA) could prevent runaway costs.

## Opportunities

### 2. Claude Design Launches with Structured CC Handoff

Anthropic Labs launched Claude Design on April 16-17, a new product for
creating visual work — designs, prototypes, slides, one-pagers — with a
direct Claude Code handoff mechanism. Powered by Opus 4.7. Available in
research preview for Pro, Max, Team, and Enterprise subscribers.

**The CC handoff mechanism:**
When a design is ready to build, Claude Design packages everything into a
handoff bundle that includes:
- Design spec (layout, spacing, typography decisions)
- Brand tokens (colors, fonts, design language)
- Component structure (identified components and hierarchy)
- Implementation notes (interactive behaviors, state management)

This bundle can be passed to Claude Code with a single instruction,
creating a closed loop from exploration to prototype to production code.

**Why OPPORTUNITY:**
The founder's Phase 6.5 screen design workflow currently flows:
`Claude.ai chat → JSX artifact → lock decisions → Visual Production Spec → CC-PROMPT package`

Claude Design's handoff bundle could tighten this to:
`Claude Design → iterate visually → handoff bundle → CC execution`

The handoff bundle is more structured than the current JSX artifact flow —
it separates concerns (spec vs. tokens vs. components vs. notes) rather
than packing everything into a prompt.

**Why not ACTION NEEDED:** Research preview. The founder needs to test
whether the handoff bundle quality matches what the current flow produces.
The current flow works — this is an improvement opportunity, not a fix for
something broken.

**Cross-system impact:**
- **Screen Design (Phase 6.5):** Primary impact. Could replace or
  complement the JSX artifact → Visual Production Spec → CC-PROMPT chain.
  The handoff bundle automates component extraction (currently manual).
- **DESIGN-SYSTEM.md template:** Claude Design's brand system integration
  could auto-apply design tokens from the project's design system.
- **Visual Production Spec:** The handoff bundle may produce equivalent
  or better specs as a structured artifact.
- **Component extraction:** Handoff bundle includes component structure —
  this step is currently part of Phase 6.5h packaging.

**Key question to test:** Take one screen from an existing project, design
it in Claude Design, generate the handoff bundle, and compare it against
what the current JSX artifact → CC-PROMPT flow would produce. Does the
bundle's component structure match what you'd write? Are the
implementation notes actionable?

## Bookmarked

### 3. /ultrareview — Multi-Agent Code Review

CC v2.1.111 (April 17) introduced `/ultrareview`, which orchestrates a
cloud fleet of reviewer agents for deep code review. Push a branch, open a
draft PR, run `/ultrareview <PR-number>`. Every finding is independently
reproduced and verified. Reviews take 5-10 minutes and run in the
background.

3 free runs per account. After that, $5-20 per review based on change size.
CC v2.1.113 improved it: faster launch with parallelized checks, diffstat
in the launch dialog, animated launching state.

**Why BOOKMARKED:** The founder's code quality assurance is Track 5 visual
QA, which is a different system focused on visual consistency. /ultrareview
adds deep code-level review — a complementary capability. At $5-20/run,
ROI depends on how often the founder merges substantial PRs solo without
external review. The 3 free runs are a no-cost way to evaluate.

### 4. Task Budgets (API Beta)

Opus 4.7 introduces task budgets — token ceilings for agentic loops. The
model sees a running countdown and finishes gracefully as the budget is
consumed. Activated via API with `task-budgets-2026-03-13` beta header.

**Why BOOKMARKED:** Currently API-only. When integrated into CC's settings
or scheduled task configuration, this could prevent runaway costs in all
agentic workflows including this scout, health checks, and CC-PROMPT
execution. Watch for CC integration.

### 5. Subagent 10-Minute Stall Timeout (v2.1.113)

CC v2.1.113 adds a 10-minute stall timeout for subagents. Previously, a
subagent that stalled mid-stream would hang silently. Now it fails with
a clear error.

**Why BOOKMARKED:** The founder's parallel-track pattern and agent teams
use subagents. A silently hanging subagent was a real reliability risk —
especially in unattended scheduled task contexts. This fix improves
parallel-track reliability without any action needed.
