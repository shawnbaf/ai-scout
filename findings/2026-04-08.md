## Action Needed

### 1. CC Default Effort Level Changed from Medium to High

CC v2.1.94 changed the default effort level from `medium` to `high` for API-key, Bedrock/Vertex/Foundry, Team, and Enterprise users. Subscription (Max plan) users retain the `medium` default. Controllable via `/effort`.

**Why this matters:** Scheduled tasks and headless sessions running under API keys or Team/Enterprise plans now consume more tokens per run by default. This scout, and any other scheduled tasks the founder runs, will be affected if they use API-key auth. The quality of output improves, but the cost increases.

**Cross-system impact:**
- **Scheduled Tasks:** This scout and any other scheduled cloud tasks will use more tokens per invocation unless explicitly set to medium effort.
- **CLAUDE.md Template:** Consider adding `/effort medium` to scheduled task prompts if cost control matters, or accept the higher default for better output quality.
- **Ops Pipeline:** Any Telegram-triggered CC sessions via Channels will also default to high effort.

**Action:** Check which auth method scheduled tasks use. If API-key or Team/Enterprise, decide whether to keep `high` (better quality) or explicitly set `medium` (lower cost). Add `/effort` setting to scheduled task prompts if needed.

Source: github.com/anthropics/claude-code/releases v2.1.94 (April 7)

## Opportunities

### 2. CC Hooks Now Support `sessionTitle` Output

v2.1.94 added `hookSpecificOutput.sessionTitle` to `UserPromptSubmit` hooks. A hook can now set the session title programmatically when a prompt is submitted.

**Why this matters:** Scheduled tasks currently get generic session titles. With this hook, each scheduled task (this scout, health checks, UI reviews) can set a descriptive title like "AI Scout — 2026-04-08" automatically, making session management and history browsing significantly easier.

**Cross-system impact:**
- **Scheduled Tasks:** Each task can self-label. The scout could set "AI Scout — [date]", health checks could set "Health Check — [project]", etc.
- **Ops Pipeline:** Telegram-triggered sessions can label themselves based on the trigger context.
- **CLAUDE.md Template:** The hooks section could document a `session-title` hook pattern using this new output field.

**Implementation:** Add a `UserPromptSubmit` hook that reads the prompt content and sets a meaningful `sessionTitle` based on the task being run.

Source: github.com/anthropics/claude-code/releases v2.1.94 (April 7)

### 3. claude-devtools — Session Observability for Autonomous Workflows

`github.com/matt1398/claude-devtools` (3K stars, free, open source). A zero-config desktop app that reads `~/.claude/` session logs and provides full execution visibility.

**Why this matters:** The founder runs autonomous scheduled tasks, health checks, and multi-agent sessions with no way to review what CC actually did. This tool shows exactly what's in CC's context window per-turn (CLAUDE.md, skills, files, tool I/O, thinking, team overhead, prompts), when and how context compresses, subagent session trees with per-agent token/cost metrics, and cross-session search.

**Cross-system impact:**
- **Scheduled Tasks:** Audit what the scout and other autonomous tasks actually did, which files were read, where tokens went.
- **Health System (Track 4):** Understand token costs per health check run.
- **UI Review (Track 5):** See how much context the route crawler consumed.
- **Cost management:** Per-session cost attribution with subagent breakdowns.

**Evaluation:** This is a review/audit tool, not a workflow driver. It runs locally, reads local files, needs no API keys or setup. It's the kind of tool that pays for itself the first time the founder wonders "what did that scheduled task actually do?" — which is every run of this scout.

Source: github.com/matt1398/claude-devtools

## Bookmarked

### 3. CC 429 Rate-Limit Fix — Headless Sessions No Longer Hang

v2.1.94 fixed agents appearing stuck after a 429 rate-limit response with a long `Retry-After` header. The error now surfaces immediately instead of silently waiting.

**Why this matters:** Headless scheduled task sessions that hit rate limits were previously hanging indefinitely with no visible error. Now they fail fast and visibly. This is a reliability improvement for all autonomous CC sessions — this scout included.

No action needed. The fix is automatic with the update to v2.1.94+.

### 4. CC Cross-Worktree Resume and Long-Session Fixes

v2.1.94 includes several quality-of-life fixes for the founder's workflow patterns:
- `--resume` now resumes sessions from other worktrees of the same repo directly, instead of printing a `cd` command. Improves the parallel worktree workflow.
- Fixed scrollback showing the same diff repeated and blank pages in long-running sessions. Affects health system runs, UI review sessions, and any extended CC session.
- Fixed CJK/multibyte text corruption in stream-json input/output.
- Fixed hyperlinks opening two browser tabs in tmux on xterm.js terminals (VS Code, Hyper, Tabby).
- Fixed alt-screen rendering ghost lines.

These are bug fixes — no action needed, all automatic with v2.1.94+.

### 5. CC Plugin System Fixes

v2.1.94 fixed several plugin-related issues:
- Plugin skills declared via `"skills": ["./"]` now use the frontmatter `name` for invocation instead of the directory basename. More stable naming across install methods.
- Plugin skill hooks defined in YAML frontmatter were being silently ignored — now work correctly.
- Plugin hooks failing with "No such file or directory" when `CLAUDE_PLUGIN_ROOT` was not set — fixed.
- `${CLAUDE_PLUGIN_ROOT}` was resolving to the marketplace source directory instead of the installed cache — fixed.

Relevant if the founder installs marketplace plugins. The skill naming change means any existing plugin invocations by directory name may need updating.

### 6. CC v2.1.96 — Bedrock Auth Regression Fix

v2.1.96 (April 8) fixed Bedrock requests failing with `403 "Authorization header is missing"` when using `AWS_BEARER_TOKEN_BEDROCK` or `CLAUDE_CODE_SKIP_BEDROCK_AUTH`. This was a regression introduced in v2.1.94. Only relevant if using Bedrock auth.
