# AI Scout Findings — 2026-03-27

## Action Needed

### 1. CC v2.1.85 — Conditional `if` field for hooks
New `if` field allows hooks to use permission rule syntax (e.g., `Bash(git *)`) to filter when they fire. Previously, hooks fired on every matching event and relied on the hook script itself to filter. This reduces process spawning overhead significantly.

**Impact:** Directly affects the founder's git-guard, auto-format, and doc-drift-reminder hooks. Each hook should be updated to use the `if` field for filtering instead of internal logic. This is a performance and clarity improvement.

**Source:** CC Changelog v2.1.85 (March 26, 2026)

### 2. CC v2.1.83 — New `CwdChanged` and `FileChanged` hook events
Two new hook event types for reactive environment management. `CwdChanged` fires when the working directory changes. `FileChanged` fires when files are modified.

**Impact:** New hook types — the founder explicitly watches for hook updates. These enable reactive workflows like auto-refreshing context when switching between projects, or triggering validation when specific files change. Could enhance the doc-drift-reminder system.

**Source:** CC Changelog v2.1.83 (March 25, 2026)

### 3. CC Auto Mode — New permission paradigm (March 24)
New permission mode between default (approve everything) and dangerously-skip-permissions. Uses a two-layer classifier system: server-side prompt-injection probe on inputs, transcript classifier (Sonnet 4.6) on outputs. Available as research preview for Team users.

**Impact:** Changes how CC permissions work. The ops pipeline CLAUDE.md section specifies explicit permission rules (allow read/write/test/git/screenshots/health/ui-review, deny prod build/migration/rm -rf/force push/push to main/merge to main). Auto mode may interact with or partially replace these rules. Needs evaluation before adoption — the founder's explicit deny rules are safety-critical.

**Source:** Anthropic engineering blog (March 24, 2026), CC Docs Changelog

## Opportunities

### 4. CC v2.1.83 — Agent `initialPrompt` frontmatter
Agents can now declare `initialPrompt` in frontmatter to auto-submit their first turn. No manual prompt needed to kick off a subagent.

**Potential:** Improves parallelization in CC-PROMPTs that use teams. Subagents can self-start, reducing orchestration overhead. Directly relevant to the "Every CC-PROMPT includes a note at the top telling CC to use teams" pattern.

**Source:** CC Changelog v2.1.83 (March 25, 2026)

### 5. CC v2.1.85 — PreToolUse hooks can satisfy AskUserQuestion
PreToolUse hooks can now return `updatedInput` with `permissionDecision: "allow"` to automatically answer AskUserQuestion prompts.

**Potential:** Could automate approval flows in the Telegram ops pipeline. When CC asks a question that has a known-safe answer, a hook can provide it automatically instead of waiting for human input.

**Source:** CC Changelog v2.1.85 (March 26, 2026)

### 6. CC v2.1.83 — `managed-settings.d/` drop-in directory
New directory for deploying independent policy fragments. Settings can be modularized into separate files.

**Potential:** "Update once, apply everywhere" — the founder's goal. Shared policy fragments could be maintained in ai-optimization-hub and symlinked or deployed across projects.

**Source:** CC Changelog v2.1.83 (March 25, 2026)

**Source:** CC Changelog v2.1.83 (March 25, 2026)
