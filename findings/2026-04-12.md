### 1. andrej-karpathy-skills — CLAUDE.md Distilled From Karpathy's LLM Coding Pitfalls

`forrestchang/andrej-karpathy-skills` is a new community plugin (surfaced April 12)
that packages Andrej Karpathy's observations on where LLMs fail at coding into a
single CLAUDE.md / skill file. It is installable as a Claude Code plugin so the
rules get auto-loaded at session start across all projects.

The behavioral guidelines it enforces:

- **Minimum code that solves the problem.** Nothing speculative, no features
  beyond what was asked.
- **Surgical changes.** Touch only what you must; clean up only your own mess.
- **Explicit assumptions.** State them up front; if uncertain, ask; if multiple
  interpretations exist, present them.
- **Goal-driven execution.** "Don't tell it what to do — give it success
  criteria and watch it go." Declarative goals with verification loops instead
  of imperative step lists.

**Why bookmarked, not opportunity:** The founder's existing CLAUDE.md Hard
Rules and Learned Corrections already cover most of this ground (the
surgical-change discipline, no-speculative-features rule, and success-criteria
pattern are baked in). This is a community-maintained distillation of the same
principles — worth reading as a cross-check, not adopting wholesale. The
interesting bit is the "success criteria, not step lists" framing, which could
sharpen how the founder writes CC-PROMPTs in Phase 6.5h.

**Cross-system impact:**
- **CLAUDE.md template:** Cross-check Hard Rules against this list. If any
  principle is sharper here than in the current template, lift the phrasing.
- **CC-PROMPT package (Phase 6.5h):** The "success criteria + verification
  loop" framing is a potential upgrade to how prompts are structured — give
  CC a verifiable target rather than a procedure.
- **Skills system:** Demonstrates the pattern of shipping a CLAUDE.md as a
  plugin so it auto-loads across projects. The founder currently copies
  CLAUDE.md per-project; plugin-shipped variant could be worth exploring for
  rules that should apply everywhere.

Source: github.com/forrestchang/andrej-karpathy-skills, aitoolly.com April 12

### 2. Anthropic Managed Agents API — Infrastructure Layer For Autonomous Agents

Anthropic's Managed Agents API (public beta, announced April 9 via Help Net
Security, covered broader April 10) provides a managed execution layer for
autonomous agents: sandboxing, state management, credential handling, tool
execution, and long-running sessions that survive disconnections. Pricing is
$0.08 per session-hour plus standard API token pricing. Access is via the
`managed-agents-2026-04-01` header on any Anthropic API account.

The feature set overlaps significantly with what Claude Code scheduled tasks
already provide for the founder's workflows — sandboxed execution, state,
credential handling, long-running sessions with disconnection recovery. The
distinction is that Managed Agents is API-level infrastructure for embedding
agents into *shipped applications*, whereas CC scheduled tasks are for
operating the founder's own dev workflows.

**Why bookmarked, not opportunity:** This doesn't replace or improve any
current workflow. Relevant only if a future founder project needs to ship
autonomous-agent features as part of the product itself (e.g., a user-facing
agent inside an app). At that point, Managed Agents would save the sandbox
+ state + credential plumbing that would otherwise need to be built.

**Cross-system impact:**
- **New project bootstrap (Phase 2 architecture):** If a future product
  includes an autonomous agent feature, Managed Agents becomes a
  decision point in architecture proposal — use it vs. roll your own
  sandbox/state.
- **Ops Pipeline (Phase 8):** Not directly affected — CC scheduled tasks
  remain the right tool for *operating* the founder's workflows.

Note: yesterday's findings covered the **Advisor Tool** (April 9, pair a fast
executor with a smarter advisor model). That is a distinct product from
Managed Agents — easy to conflate since both launched the same week. Neither
directly changes the founder's current workflow.

Source: helpnetsecurity.com April 9, siliconangle.com April 8, unite.ai April 2026
