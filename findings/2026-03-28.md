## Opportunities

- **New hook events: `CwdChanged`, `FileChanged`, `TaskCreated`** (CC v2.1.83-84, March 25-26)
  - `CwdChanged` fires when working directory changes (useful for direnv-style env management).
  - `FileChanged` fires when files are modified — could make doc-drift-reminder trigger only on specific file changes rather than all edits.
  - `TaskCreated` fires when a scheduled task is created via `TaskCreate` — enables automation chains (e.g., notify on Telegram when a new task spins up).
  - Affects: `new-project-master.md` → Health System hooks, ops pipeline automation

- **PreToolUse hooks can auto-answer `AskUserQuestion`** (CC v2.1.85, March 26)
  - Returning `updatedInput` alongside `permissionDecision: "allow"` from a PreToolUse hook can satisfy `AskUserQuestion` calls.
  - This enables fully headless integrations — scheduled tasks and channels sessions can auto-respond to prompts without human intervention.
  - Affects: Scheduled task reliability, Phase 8 ops pipeline headless operation

  - `allowedChannelPlugins` managed setting added in v2.1.84 for enterprise plugin control.
  - `AskUserQuestion` disabled in channels mode (v2.1.83) to prevent blocking on unanswerable prompts.
  - The Telegram permission relay from phone is becoming production-viable. Directly relevant to Phase 8 ops pipeline.
  - Affects: `new-project-master.md` → Phase 8 Ops Pipeline, Telegram remote management
