## Opportunities

### 1. cc-lens — Local CC Analytics Dashboard (`npx cc-lens`)

Zero-install local dashboard that reads `~/.claude/` directly. Shows token
usage by project, session history, cost estimates per model, tool usage
rankings, cache efficiency panel, activity heatmap, and recent sessions.
No cloud, no telemetry, no API key. Runs via `npx cc-lens` — opens a local
web app in the browser.

Per-project cards display sessions, duration, estimated cost, languages,
git branches, MCP/agent badges, and top tools. Includes per-model token
and cost breakdown.

**Workflow impact:** The founder runs CC heavily across multiple projects
and explicitly watches costs. This gives per-project cost breakdowns
without any setup — just `npx cc-lens`. Supports the "one command not
scattered scripts" philosophy. CLI-launchable, free, local-only.

**Evaluation against templates:**
- Does this change an existing workflow? No.
- Does this enable something new? Yes — per-project cost visibility across
  all CC sessions. Informs effort-level decisions and project prioritization.
- Does this obsolete a workaround? No specific workaround, but fills a
  monitoring gap.
- Founder profile check: CLI-drivable, free tier, no team required, no cloud
  dependency. Passes all "Signal" filters.

**Cross-system impact:**
- Could inform which projects consume the most tokens, guiding effort-level
  management and `/model` routing decisions.
- Not a template change — no file updates needed.

## Bookmarked

### 2. `ant` CLI — Anthropic's Official API CLI

Anthropic shipped an official Go-based CLI for the Claude API
(`github.com/anthropics/anthropic-cli`). Resource-based command structure:
`ant [resource] <command> [flags]`, like kubectl for Claude. Install via
`brew install anthropics/tap/ant` (macOS) or `go install`.

Features: typed flags or piped YAML (no hand-written JSON), `@path` file
references for inlining file contents, `--transform` for field extraction.
Beta resources (agents, sessions, deployments, environments, skills) live
under the `beta:` prefix.

**Workflow impact:** The founder primarily uses CC, not the API directly.
But for automation scripts, hooks, or scheduled task tooling, having an
official CLI could be useful. The beta resource management (agents,
sessions) overlaps with CC's territory and may become relevant as
Anthropic's agent infrastructure evolves.

**Why bookmarked, not opportunity:** The founder's current automation runs
through CC itself. The ant CLI would only add value for workflows that
need direct API access outside of CC sessions — not the current pattern.
Worth knowing about and revisiting if the automation needs change.

### 3. CC v2.1.126 Uncovered Detail — Gateway Model Picker

Yesterday's sweep covered v2.1.126 but missed this detail: the `/model`
picker now lists models from your gateway's `/v1/models` endpoint when
`ANTHROPIC_BASE_URL` points at an Anthropic-compatible gateway. This means
if the founder runs Ollama behind a proxy that implements the Anthropic
API format, CC's model picker would discover and list those local models
automatically.

**Why bookmarked:** Requires setting up an Anthropic-compatible proxy in
front of Ollama first. Useful when the local model routing setup is ready,
but not immediately actionable.
