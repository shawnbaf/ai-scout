### 1. CC v2.1.101 — Command Injection Fix + Subagent Isolation Bugs + Hook Settings Hardening

Three critical categories of fixes in v2.1.101 (April 10) that directly affect the founder's workflows:

- **Command injection vulnerability** in POSIX `which` fallback for LSP binary detection. A security fix — any CC session using LSP-connected editors was potentially exposed.
- **Sub-agents in isolated worktrees denied Read/Edit access to their own worktree.** This means parallel agents using worktree isolation were unable to read or edit files in their own worktree — silently breaking the isolation model the founder relies on for multi-agent workflows.
- **Subagents not inheriting MCP tools from dynamically-injected servers.** If MCP servers were added mid-session (or via dynamic injection), subagents couldn't use those tools. This affects any multi-agent workflow where the parent session sets up MCP connections.
- **Settings resilience: unrecognized hook event names no longer cause entire settings.json to be ignored.** Previously, a single typo in a hook event name (e.g., `PreTooluse` instead of `PreToolUse`) would silently disable the entire settings file — meaning hooks, permissions, and all settings would stop working without any error message.

Additional notable fixes in v2.1.101:
- Memory leak in virtual scroller retaining historical message copies (affects long sessions)
- `--resume`/`--continue` losing context on large sessions
- `--resume` chain recovery bridging into unrelated subagent conversations
- Crash on `--resume` when persisted Edit/Write tool result was missing `file_path`
- Hardcoded 5-minute request timeout aborting slow backends (now configurable)
- `permissions.deny` rules not overriding PreToolUse hook's `permissionDecision: "ask"`
- `claude -w <name>` failing with "already exists" after stale directory
- Sandboxed Bash `mktemp` failures after fresh boot
- Multiple plugin issues (slash commands resolving incorrectly, stale cache, skills not honoring frontmatter)
- Remote Control issues (worktrees removed on crash, connection failures)
- 31 bug fixes total

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** Subagent worktree + MCP inheritance fixes restore parallel agent isolation integrity. Remote Control fixes improve remote session stability.
- **Hooks:** Settings resilience fix means the git-guard, auto-format, and doc-drift-reminder hooks can't be silently disabled by an unrecognized event name elsewhere in settings.json.
- **Multi-agent workflows:** Both worktree isolation and MCP tool inheritance were broken — any recent parallel agent work should be considered potentially compromised.
- **Security:** Command injection fix in LSP binary detection. Update immediately.
- **This scout:** Runs as a scheduled task with MCP servers — affected by the MCP inheritance bug.

**Action:** Update CC to v2.1.101 immediately. Verify all hooks are active post-update. Re-test parallel worktree workflows.

## Opportunities

### 3. Advisor Tool Public Beta — Smarter Agentic Workloads

Anthropic launched the Advisor Tool in public beta (April 9). It enables developers to "pair a faster executor model with a higher-intelligence advisor model that provides strategic guidance mid-generation" for agentic workloads.

**How it works:** During agentic execution, the executor model (fast, cheap) handles routine operations. When it hits a decision point requiring deeper reasoning, it can consult the advisor model (slower, smarter) for strategic guidance. The advisor provides direction without taking over execution.

**Why it matters:** This pattern could improve the quality of autonomous agent sessions (like this scout or other scheduled tasks) without the full cost of running the smartest model for every turn. The executor handles file reads, web fetches, and routine processing; the advisor handles evaluation decisions and strategic categorization.

**Cross-system impact:**
- **Scheduled tasks:** Could run this scout with a fast executor + smart advisor, reducing cost while maintaining evaluation quality.
- **Multi-agent workflows:** Advisor pattern could coordinate multiple executor agents.
- **Ops Pipeline:** Build/dispatch/heal modes could use advisor for triage decisions.

Source: releasebot.io/updates/anthropic (April 9)

### 5. 16+ Parallel Agent Scaling — Flat Coordination Beats Sequential

Anthropic's engineering blog documented building a C compiler with 16+ parallel Claude Code agents, producing 100,000 lines across nearly 2,000 sessions. Key scaling insight: **agent teams (flat, peer-based parallel work) outperform sub-agents (hierarchical, sequential dependency chains) at agent counts above 8.**

**The pattern:**
- Below 8 agents: hierarchical sub-agents work fine for dependency chains
- Above 8 agents: flat coordination with agent teams produces better throughput
- The bottleneck shifts from task completion to coordination overhead
- Peer agents with clear boundaries and shared context (via worktrees or shared specs) scale more predictably

**Cross-system impact:**
- **Multi-agent workflows:** The founder's current approach uses both sub-agents and agent teams. For large parallelizable tasks (e.g., Phase 7 health system setup across multiple tracks), flat coordination may produce better results.
- **Project bootstrap:** Phase 1-6 has natural dependency chains (sequential). Phase 7 tracks (Sentry, feedback, health CLI, UI review) are independent — this is where 8+ flat agents would shine.
- **CLAUDE.md:** CC-PROMPT parallelization notes could reference the 8-agent threshold for choosing teams vs. sub-agents.

Source: anthropic.com/engineering/building-c-compiler

### 6. PostToolUse Hook — Auto-Run Tests on Every File Edit

Community pattern emerging: configure a `PostToolUse` hook on `Write|Edit` that auto-runs the project's test suite (or relevant subset) after every file modification during agent sessions. This creates an immediate regression feedback loop — the agent sees test failures within seconds of making a change, before moving to the next file.

**How it differs from existing hooks:** The founder already has `auto-format` as a PostToolUse hook on Write/Edit. This extends the pattern to include test execution. The key is running only *relevant* tests (file-matched or module-scoped), not the full suite.

**Cross-system impact:**
- **Health System (Track 4):** Catches regressions during development, not just during health checks.
- **Hooks:** New hook pattern alongside git-guard, auto-format, doc-drift-reminder. Would be the fourth standard hook.
- **CLAUDE.md:** Could document as an optional hook for projects with fast test suites.

Source: community patterns via web search (multiple sources)
