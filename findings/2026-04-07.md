## Opportunities

### 1. [CATCH-UP] CC Computer Use in CLI — Native App Verification from Terminal

Computer use arrived in the CLI as a research preview in v2.1.86 (March 30, Week 14 digest). This was missed in previous findings — the March 29 sweep covered computer use in the Desktop app, but the CLI extension was not captured.

**What it does:** Claude can now open native apps, click through UI, test its own changes, and fix what breaks — all from the terminal. Enable via `/mcp` → toggle `computer-use`. Example: "Open the iOS simulator, tap through onboarding, and screenshot each step."

**Why this matters more than desktop computer use:** The founder's workflow is terminal-first. Desktop computer use required the Desktop app; CLI computer use works in any terminal session, including SSH, tmux, and scheduled tasks. This closes the GUI verification gap for native apps that Playwright can't reach.

**Cross-system impact:**
- **UI Review (Track 5):** The route crawler uses Playwright for web app screenshots. Computer use could extend visual verification to native iOS/macOS apps or Electron apps where Playwright has limited reach.
- **Demo Navigator (Phase 9):** For apps with native components, computer use could verify demo flows that currently require manual testing.
- **Demo Video (Phase 10):** Could supplement ghost-cursor-playwright for recording interactions with native UI elements. However, Playwright still gives more precise programmatic control for web recording — computer use is complementary, not a replacement.
- **Health System (Track 4):** CC could visually verify that native app builds work correctly as part of health checks.
- **Scheduled Tasks:** CLI computer use means autonomous sessions can do GUI verification without the Desktop app.

**Limitations:** Still in research preview — expect rough edges. Best for closing the loop on things only a GUI can verify. Playwright remains the better tool for web-based automation where precise selectors and programmatic control matter.

**Action:** Test computer use in a local CC session: `/mcp` → enable `computer-use` → ask CC to open a browser, navigate to a dev URL, and screenshot. Evaluate quality and reliability before integrating into any automated workflow.
