## Action Needed

### 1. CC v2.1.121 Fixes Multi-GB Memory Leaks and Resume Crashes — Update Immediately

v2.1.121 (39 changes) fixes several stability issues that affect long-running
sessions and the founder's workflow:

- **Unbounded memory growth (multi-GB RSS)** when processing many images —
  directly affects UI review sessions processing screenshots.
- **`/usage` leaking ~2GB of memory** on large transcript histories — affects
  any long session.
- **`--resume` crashing on startup** in external builds.
- **`--resume` failing on corrupted transcript lines** from unclean shutdowns.
- **Scrollback duplication** on tmux, GNOME Terminal, Windows Terminal, Konsole.
- **MCP servers auto-retry up to 3 times** on transient startup errors — fixes
  reliability for Composio and GitHub MCP connections.
- **"Always allow" rules for built-in tools in remote sessions** now survive
  worker restarts.
- **`NO_PROXY` now respected** for all HTTP clients in managed-settings.
- **Microsoft 365 MCP OAuth** fixed (duplicate `prompt` parameter issue).

v2.1.120 also shipped (April 24-25) but had a startup crash bug that triggered
auto-rollback to v2.1.119 for some users. The fixes in v2.1.120 are carried
forward into v2.1.121.

**Cross-system impact:**
- **All CC sessions:** Update to v2.1.121 immediately. Memory leak fixes alone
  justify the update — multi-GB RSS growth was silently degrading long sessions.
- **UI Review (Track 5):** Image processing memory fix prevents OOM during
  screenshot-heavy review sessions.
- **Ops Pipeline:** MCP auto-retry makes Composio and GitHub MCP connections
  more reliable in scheduled tasks.
- **Remote sessions:** "Always allow" persistence fix means permission rules
  stay consistent across worker restarts.

## Opportunities

### 2. `alwaysLoad` MCP Config Pins Critical Tools — No More Deferred Loading

v2.1.121 adds an `alwaysLoad` option for MCP server configuration. Tools from
servers marked `alwaysLoad` skip the deferral/Tool Search step and are always
available in the context window.

This directly affects scheduled tasks and multi-MCP sessions where specific
tools must be immediately available without waiting for Tool Search resolution.
Currently, deferred tools require an explicit `ToolSearch` call before use —
`alwaysLoad` eliminates this for critical MCP servers.

**Cross-system impact:**
- **Scheduled tasks:** Pin GitHub MCP and Composio as `alwaysLoad` so tools
  are available from the first turn without Tool Search.
- **CLAUDE.md template:** Document `alwaysLoad` option in MCP server config.
- **new-project-master.md:** Add to Phase 2 architecture as MCP config option.

### 3. PostToolUse Hooks Now Replace Output for ALL Tools

v2.1.121 expands `hookSpecificOutput.updatedToolOutput` in PostToolUse hooks
to work with all tools, not just MCP tools. Previously this was MCP-only.

This means hooks can now intercept and modify the output of any tool call —
Read, Write, Edit, Bash, etc. Opens automation patterns like:
- Annotating Bash output with warnings or additional context
- Filtering sensitive data from Read output
- Adding metadata to Edit confirmations

**Cross-system impact:**
- **Hooks (all projects):** The founder's existing hooks could use output
  replacement for richer feedback. git-guard could add context to blocked
  commands. auto-format could report what changed.
- **CLAUDE.md template:** Document the expanded output replacement capability.

### 4. `ultrareview` CLI Subcommand — Run Code Reviews from CI and Scripts

v2.1.120 adds `claude ultrareview [target]` as a non-interactive subcommand.
Previously `/ultrareview` was only available interactively. Now it can be
triggered from CI pipelines, scripts, and scheduled tasks.

**Cross-system impact:**
- **Ops Pipeline:** Scheduled tasks could run ultrareview on PRs automatically.
- **CLAUDE.md template:** Document as a CI integration option.

### 5. Playwright Screencast API — Chapter Markers and Annotations for Demo Recording

Playwright v1.59.0 (shipped April 1, missed in prior sweeps) introduced
`page.screencast` — a purpose-built API for agentic video recording:

- **Action annotations:** Automatically highlights interacted elements during
  recording.
- **Chapter markers:** `showChapter()` and `showOverlay()` for segmenting
  recordings into labeled sections.
- **Real-time JPEG frame streaming:** Enables AI vision or live preview of
  recordings in progress.
- **Agentic video receipts:** Annotated walkthrough videos produced after
  task completion.

Also in v1.59.0:
- `browser.bind()` — lets a launched browser be connected to by playwright-cli
  and other clients.
- `npx playwright test --debug=cli` — coding agents can attach and debug live.
- `page.ariaSnapshot()` — accessibility snapshot for ARIA tree inspection.

**Cross-system impact:**
- **Demo Video (Phase 10):** Chapter markers and action annotations could
  replace ghost-cursor-playwright for demo recording. The frame stream could
  feed Remotion compositions directly. This is potentially a pipeline
  architecture change.
- **Demo Navigator (Phase 9):** screencast could auto-generate walkthrough
  videos from demo sequences without separate recording scripts.
- **UI Review (Track 5):** `browser.bind()` improves agent-to-browser attachment
  for the route crawler. `page.ariaSnapshot()` adds accessibility audit
  capability.
- **Launch Checklist:** `page.ariaSnapshot()` enables automated accessibility
  verification.
- **project-audit.md:** Add screencast-based recording as an option alongside
  ghost-cursor-playwright.

### 6. Community Architecture Invariant Registry — Hooks as Hard Gates

A well-documented community repo (vinicius91carvalho/.claude) implements hooks
as deterministic enforcers rather than advisory checks:

- **Architecture Invariant Registry:** Cross-module contracts verified post-edit.
  Hooks check that module boundaries aren't violated, imports stay within
  allowed dependency paths, etc.
- **TDD gates:** Edit hooks block code changes that don't have corresponding
  tests, enforced via hook exit codes (not CLAUDE.md instructions that can
  be ignored).
- **Cross-project error registry:** Shared error patterns learned from one
  project applied to all projects.
- **Orchestrator/executor/reviewer agent split** using worktrees for isolation.

Key insight: hooks are deterministic (exit code 2 = blocked) while CLAUDE.md
instructions are advisory. Moving critical invariants from CLAUDE.md to hooks
makes them unbypassable.

**Cross-system impact:**
- **Hooks (all projects):** The founder's git-guard already uses this pattern
  for branch protection. Extending it to architecture invariants (import
  boundaries, test requirements) would strengthen the enforcement layer.
- **CLAUDE.md template:** Could move some Hard Rules to hooks where
  deterministic enforcement is more appropriate than advisory instructions.
- **project-audit.md:** Add "architecture invariant hooks" as an optional
  check item.

experiment: Opus agents closed ~2 more deals per participant than Haiku agents
