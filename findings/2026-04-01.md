## Action Needed

### 1. Claude Code v2.1.89 — "defer" Permission for PreToolUse Hooks

New hook capability: PreToolUse hooks can now return a "defer" decision in headless (`-p`) sessions. The session pauses at the tool call and can be resumed later with `-p --resume`, allowing the hook to re-evaluate. This fundamentally changes how scheduled tasks handle permission boundaries.

**Why it matters:** Currently, scheduled tasks must either pre-approve all tool calls or fail on unapproved ones. With defer, a scheduled task can pause at a sensitive operation (e.g., destructive git command, file deletion) and wait for human review before continuing. This is the missing piece for "supervised autonomy" in cloud tasks.

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** Scheduled tasks can now implement approval gates without failing
- **Hooks:** New decision type alongside allow/deny — update hook documentation
- **CLAUDE.md:** Document defer as a hook capability in scheduled task sections

**Action:** Update hooks in scheduled tasks to use defer for high-risk operations instead of blanket allow/deny.

Source: code.claude.com/docs/en/changelog (v2.1.89, April 1, 2026)

## Opportunities

### 3. CC v2.1.89 — MCP Non-Blocking + Faster `-p` Startup

Two related improvements for headless/scheduled mode:
- `MCP_CONNECTION_NONBLOCKING=true` for `-p` mode skips MCP connection wait entirely
- `--mcp-config` server connections now bounded at 5s instead of blocking on the slowest server
- Unauthenticated HTTP/SSE MCP servers save ~600ms on startup

Combined effect: scheduled tasks using MCP servers (like Composio for Telegram) start significantly faster and don't hang on slow/unreachable MCP connections.

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** Faster scheduled task startup, more resilient to MCP failures
- **This scout:** If using MCP connectors, startup is faster and won't hang

Source: code.claude.com/docs/en/changelog (v2.1.89)

### 4. CC v2.1.89 — Non-Streaming Token Cap Tripled (21k → 64k)

Non-streaming fallback token cap increased from 21,000 to 64,000 tokens, with timeout extended from 120s to 300s (local). This affects `claude -p` usage and any SDK integrations using non-streaming responses.

**Why it matters:** Scheduled tasks that produce long outputs (like this scout's findings) were previously capped at 21k tokens in non-streaming mode. The 3x increase means complex single-response tasks are less likely to truncate.

Source: code.claude.com/docs/en/changelog (v2.1.89)

### 5. CC v2.1.89 — Performance: WASM yoga-layout Replaced with Pure TypeScript

The WASM yoga-layout dependency was replaced with a pure TypeScript implementation. Improves scroll performance on large transcripts and reduces UI stutter when compaction triggers.

**Why it matters:** Long CC sessions (like multi-hour development sessions or extended scheduled tasks) had visible stutter during compaction. This is a direct quality-of-life improvement for heavy CC users.

Source: code.claude.com/docs/en/changelog (v2.1.89)
