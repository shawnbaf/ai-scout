## Action Needed

### 1. Claude Code v2.1.88 Released (March 30)

The most significant CC release this month. Multiple changes directly affect the founder's workflow:

**New PermissionDenied hook:**
- Fires after auto mode classifier denies a command. Return `{retry: true}` to let the model retry.
- Enables automated recovery from permission denials in scheduled tasks and autonomous sessions.
- This is a new hook event type — extends the hooks system (git-guard, auto-format, doc-drift-reminder).
- Affects: `project-audit.md` → Hooks section, scheduled task reliability.

**CLAUDE.md re-injection bug fixed:**
- Nested CLAUDE.md files were being re-injected dozens of times in long sessions that read many files.
- This was silently eating context budget, explaining why long sessions degrade.
- Validates Boris Cherny's "lean CLAUDE.md" advice (reported yesterday).
- Affects: All long-running CC sessions, this scout's own session stability.

**Prompt cache miss fix:**
- Tool schema bytes changing mid-session caused prompt cache misses.
- Combined with the CLAUDE.md fix, long-running scheduled tasks should now maintain better cache efficiency.
- Affects: Scheduled tasks, long sessions, token consumption.

**Hooks fixes:**
- `if` condition now correctly matches compound commands (`ls && git push`) and env-var prefixed commands. The git-guard hook may now catch commands it was previously missing.
- PreToolUse/PostToolUse hooks now provide `file_path` as absolute path for Write/Edit/Read tools. The auto-format hook benefits from this correctness fix.
- Affects: `project-audit.md` → Hooks (git-guard, auto-format).

**Other notable changes:**
- Named subagents in `@` mention typeahead — better multi-agent orchestration UX.
- `CLAUDE_CODE_NO_FLICKER=1` env var — flicker-free alt-screen rendering.
- StructuredOutput ~50% failure rate fixed (schema cache bug with multiple schemas).
- Memory leak fix (large JSON inputs retained as LRU cache keys).
- OOM crash fix for Edit on files >1 GiB.
- `/stats` now counts subagent/fork token usage.
- SDK error results now correctly set `is_error: true`.
- 27 total bug fixes, 3 new features, 6 behavioral changes.

**Action:** Update CC to v2.1.88. Evaluate adding a PermissionDenied hook for scheduled tasks.

Sources: github.com/anthropics/claude-code/blob/main/CHANGELOG.md, code.claude.com/docs/en/changelog, releasebot.io/updates/anthropic/claude-code

### 2. npm Security: Malicious axios Package During CC Update Window (March 31)

CC's source code was leaked via a `.map` file in the npm registry (second time — first was early 2025). During the window around the v2.1.88 publish (March 31, 00:21-03:29 UTC), a malicious `axios` package (v1.14.1 or v0.30.4) containing a RAT was briefly available on npm.

**Action:** If CC was updated via npm on March 31 during those hours, verify `axios` version in node_modules. Run `npm ls axios` in the CC install directory to check.

The leak also revealed upcoming CC features (see Bookmarked section for KAIROS and ULTRAPLAN).

Sources: venturebeat.com, cybersecuritynews.com, news.ycombinator.com/item?id=47584540

### 3. Peak-Hour CC Rate Limits Now Permanent (March 26+)

Anthropic officially confirmed intentional throttling during peak hours (5:00-11:00 AM Pacific, weekdays). Each message burns ~1.5-2x more of the 5-hour session budget during peak hours. The off-peak promotion (doubled limits) ended March 28. ~7% of users affected. API customers are NOT affected (billed per-token).

**Community workarounds:**
- Shift intensive tasks to off-peak hours
- Break large projects into shorter discrete sessions (cache invalidation compounds over long conversations)
- Avoid `--resume` during peak hours

**Action:** Verify scheduled cloud tasks don't run during 5-11am PT. Consider API access for automated workflows like this scout if subscription limits become a bottleneck.

Sources: theregister.com/2026/03/31/anthropic_claude_code_limits, news.ycombinator.com/item?id=47535897, roborhythms.com

### 7. Addy Osmani's "Code Agent Orchestra" Patterns

Google Cloud AI director published comprehensive multi-agent guide with concrete patterns:

- **Ralph Loop**: Stateless-but-iterative loop per agent — pick task, implement, validate, commit if pass, reset context, repeat. Avoids context overflow via external memory.
- **Token Budgeting**: Hard per-agent budgets (e.g., Frontend 180k, Backend 280k). At 85%, auto-pause and notify lead. 3+ iterations on same error → kill and reassign fresh.
- **REFLECTION.md**: After every task, agent writes what surprised it, patterns to add, prompt improvements. Lead reviews and merges approved learnings. More structured than Boris's `lessons.md`.
- **Key insight**: "Three focused agents consistently outperform one generalist working three times as long." Verification, not generation, is now the bottleneck.

- Cross-system: affects CC-PROMPT parallelization notes, subagent orchestration patterns, scheduled task design.

Source: addyosmani.com/blog/code-agent-orchestra/
