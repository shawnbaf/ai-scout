## Action Needed

### 1. CC v2.1.92 — Stop Hook Semantics Fix (Breaking for Scheduled Tasks)

CC v2.1.92 (April 4) fixes a bug where prompt-type Stop hooks incorrectly failed when the small fast model returned `ok:false`. This restored `preventContinuation:true` semantics for non-Stop prompt-type hooks. If the founder's scheduled tasks or hook chains use prompt-type Stop hooks for quality gates, they may have been silently misbehaving since the regression was introduced.

**Cross-system impact:**
- **Ops Pipeline (Phase 8):** Any scheduled task using Stop hooks for quality enforcement was potentially affected — hooks that should have prevented continuation may have let bad output through.
- **Hooks:** The founder's git-guard, auto-format, and doc-drift-reminder hooks should be tested on v2.1.92 to verify correct behavior.

**Action:** Update CC to v2.1.92. Test any prompt-type Stop hooks to confirm they correctly block continuation.

Source: code.claude.com/docs/en/changelog (v2.1.92, April 4)

### 2. CC v2.1.92 — Subagent Spawning Fix After tmux Window Changes

Subagent spawning permanently failed with "Could not determine pane count" after tmux windows were killed or renumbered during long-running sessions. This directly affects worktree-based parallel agent workflows where tmux sessions are the orchestration layer.

**Cross-system impact:**
- **Parallel Agents:** The founder runs multiple CC agents via tmux + worktrees. If a tmux window was closed during a long session, all subsequent subagent spawns in that session would fail permanently. This was a silent reliability issue.

**Action:** Update to v2.1.92. This fix is critical for anyone running parallel agents in tmux.

Source: code.claude.com/docs/en/changelog (v2.1.92, April 4)

## Opportunities

### 3. CC v2.1.92 — Per-Model and Cache-Hit Breakdown in /cost

The `/cost` command now shows detailed cost metrics broken down by model and cache performance. Previously you could see total cost but not which model was burning tokens or how effective prompt caching was.

**Why it matters:** The founder can now see exactly how much each model costs per session and whether prompt caching is working. Combined with the new prompt cache expiration footer hint (shows uncached token count when returning to a stale session), this gives real cost visibility for optimizing session management.

**Cross-system impact:**
- **Ops Pipeline:** Scheduled tasks can be evaluated for cost efficiency per-model.
- **Session management:** Footer hint tells you when resuming a session will be expensive because the cache expired.

Source: code.claude.com/docs/en/changelog (v2.1.92, April 4)

### 4. claude-devtools — Session Observability Without Modifying CC

Desktop app that reads `~/.claude/` session logs and reconstructs full execution details: per-turn token attribution across 7 categories (CLAUDE.md, skills, @-mentions, tool I/O, extended thinking, team coordination, user prompts), compaction boundary visualization, recursive subagent execution trees, and custom alerts for `.env` access or high token usage.

**Why it matters:** The founder runs complex multi-agent sessions with skills, hooks, and subagents. This tool shows what's actually consuming context — is it CLAUDE.md overhead? Skill activations? Tool output bloat? Compaction visualization reveals when and how CC silently compresses conversations, which is invisible in the terminal. No setup, no API keys, reads existing logs.

**Cross-system impact:**
- **All systems:** Diagnose context window pressure across any workflow. See which skills or tools are consuming disproportionate context.
- **Scheduled tasks:** Post-mortem analysis of autonomous sessions — did it compact? Where did context go?
- **Parallel agents:** Subagent tree visualization shows the full orchestration topology.

Source: github.com/matt1398/claude-devtools

### 5. Claude Octopus — Multi-Model Audit via Ollama + External Providers

CC plugin that puts up to 8 AI models on every coding task to catch blind spots. Works as a CC plugin (`/octo:` command prefix), supports Ollama for free local model auditing, and activates only when invoked. Runs models in parallel for research, sequentially for problem-solving, and adversarially for review.

**Why it matters:** The founder has Ollama on the Alienware (8GB VRAM). This plugin can use the local Ollama instance as a free second opinion on CC output — essentially getting a Gemma/Llama review of Claude's code without leaving CC. Zero API cost for the local model path. Detects installed providers automatically.

**Cross-system impact:**
- **Local model routing:** Gives the Alienware's Ollama a practical use case beyond standalone inference — it becomes a quality gate within CC itself.
- **Code quality:** Multi-model disagreements surface edge cases a single model misses.

Source: github.com/nyldn/claude-octopus (v9.19.2, 914 commits, 146 tests)
