## Opportunities

- **TurboQuant published — 4-6x KV cache compression for local models** (March 24, 2026 — ICLR 2026 paper by Google Research)
  - This is exactly what the founder has been monitoring. TurboQuant compresses KV cache to 3-4 bits per element with negligible quality loss. No retraining, no fine-tuning, no calibration required.
  - Combines two techniques: **PolarQuant** (polar coordinate decomposition, eliminates per-block normalization) and **QJL** (1-bit error correction).
  - At 3.5 bits on Llama-3.1-8B-Instruct: LongBench score 50.06, identical to 16-bit baseline. At 4 bits: up to 8x speedup in attention logits on H100.
  - **llama.cpp integration status**: Feature branch exists at `github.com/mudler/llama.cpp/tree/feat/turbo-quant`. CPU implementation (C, no deps) has 18/18 tests passing, MSE matches paper. CUDA kernels written for GPU quantize/dequant and fused attention. 6-phase integration spec exists. Community targeting llama.cpp Q3 2026; Google official Q2 2026.
  - **Practical meaning for 8GB VRAM**: A 4x KV cache reduction means roughly 4x longer context windows within the same VRAM envelope. A model that currently fits with 4K context could potentially run at 16K context. 4-bit is the sweet spot; 3-bit degrades on models <8B.
  - Affects: `CLAUDE.md` → "TurboQuant/PolarQuant PRs in llama.cpp" watch item, local model routing on Alienware
  - Sources: research.google/blog/turboquant, github.com/ggml-org/llama.cpp/discussions/20969, github.com/ggml-org/llama.cpp/issues/20977
