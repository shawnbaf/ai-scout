- **Playwright 1.58 removed `_react` and `_vue` selectors** (March 2026)
  - Breaking: `_react` and `_vue` selectors are fully removed. Also removed: `:light` selector suffix and `Devtools` launch option. Dropped macOS 13 support for WebKit.

  - Sources: playwright.dev/docs/release-notes, github.com/microsoft/playwright/releases

- **TurboQuant KV cache compression — llama.cpp implementation passing tests** (March 24, 2026 — ICLR 2026)
  - The founder explicitly monitors for this (`CLAUDE.md` → "TurboQuant/PolarQuant PRs in llama.cpp"). Published by Google Research. Compresses KV cache to 3-4 bits per element — **4-6x reduction in KV cache memory** with negligible quality loss. No retraining required.
  - Combines **PolarQuant** (polar coordinate decomposition, eliminates per-block normalization) and **QJL** (1-bit error correction, though community finds MSE-only wins in practice).
  - 4-bit is the sweet spot: quality indistinguishable from FP16 on 3B+ models. At 3.5 bits on Llama-3.1-8B-Instruct: LongBench score 50.06, identical to 16-bit baseline.
  - **llama.cpp integration status**: Working CPU implementation (C, no deps, 18/18 tests passing). CUDA kernels written but awaiting GPU validation. 6-phase integration spec filed. TQ4 (4-bit): 3.8x compression vs FP16. Official release expected Q2 2026.
  - **Practical meaning for 8GB VRAM**: A model that currently fits with 4K context could potentially run at ~16K context within the same VRAM envelope.
  - Competitor: NVIDIA KVTC claims 20x compression but requires per-model calibration.
  - Affects: Local model routing on Alienware, context window expansion for local inference
  - Sources: research.google/blog/turboquant, github.com/ggml-org/llama.cpp/discussions/20969, github.com/ggml-org/llama.cpp/issues/20977

- **context-mode plugin — 98% context savings on tool outputs** (March 2026, MIT license)
  - MCP server (`mksglu/context-mode`) intercepts tool outputs, processes in sandboxed subprocesses, returns only summaries. 315KB raw output → 5.4KB. Sessions extend from ~30min to ~3hrs.
  - Session continuity via SQLite FTS5 — when conversation compacts or resumes with `--continue`, working state rebuilds automatically.
  - **Different from MCP lazy loading** (bookmarked yesterday) — lazy loading reduces tool *schema* loading, Context Mode reduces tool *output* consumption. They're complementary.
  - Install: `/plugin marketplace add mksglu/context-mode`
  - Affects: All scheduled tasks, long CC sessions, this scout's own session longevity
  - Source: github.com/mksglu/context-mode
