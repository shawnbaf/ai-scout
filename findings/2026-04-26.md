## Action Needed

### 1. Quality Postmortem Confirms Three Bugs — All Fixed, Limits Reset

Anthropic published an engineering postmortem on April 23 disclosing three
changes that degraded Claude Code quality:

1. **Effort downgrade (Mar 4 → Apr 7):** Default reasoning effort changed
   from high to medium to reduce latency. Reverted after user reports.
2. **Thinking cache bug (Mar 26 → Apr 10):** A change to clear old thinking
   from idle sessions had a bug that kept firing every turn, making Claude
   seem forgetful and repetitive in resumed sessions.
3. **Verbosity system prompt (Apr 16 → Apr 20):** A prompt instruction to
   reduce verbosity hurt coding quality in combination with other changes.

All three resolved as of v2.1.116 (April 20). Anthropic reset usage limits
for all subscribers as compensation. If the founder experienced degraded
quality during March–April, it was confirmed real and is now fixed.

Additionally, v2.1.118 and v2.1.119 fix bugs directly relevant to the
founder's workflow:
- **Skills re-execution after compaction** — skills invoked before
  auto-compaction were being re-executed against the next user message. Fixed
  in v2.1.119. This directly affects the self-growing skills system.
- **Auto mode overriding plan mode** — auto mode was inserting conflicting
  "Execute immediately" instructions that overrode plan mode. Fixed in
  v2.1.119.
- **Stale worktree reuse** — `Agent` tool with `isolation: "worktree"` was
  reusing worktrees from prior sessions. Fixed in v2.1.119. Affects parallel
  agent workflows.
- **Plan mode not acting on existing plan** — `/plan` and `/plan open` were
  not engaging the existing plan. Fixed in v2.1.119.

**Cross-system impact:**
- **All CC sessions:** Update to v2.1.119 immediately.
- **Skills system:** The re-execution bug means skills may have been running
  twice. Verify skill behavior after updating.
- **CLAUDE.md template:** The effort level is now `high` by default (from
  v2.1.117) and the postmortem confirms this is intentional. No need to
  explicitly set effort unless optimizing for cost.

## Opportunities

### 2. Hooks Can Invoke MCP Tools Directly — New "mcp_tool" Type

CC v2.1.118 adds `type: "mcp_tool"` as a hook handler type. Hooks can now
call tools on already-connected MCP servers without shelling out. Combined
with the existing `type: "prompt"` (single-turn LLM evaluation) and
`type: "agent"` (multi-turn verification with tool access) handler types,
the hook system now supports four handler types: command, http, mcp_tool,
prompt, and agent.

This opens automation patterns like:
- PostToolUse hook → call GitHub MCP to create an issue when a TODO is added
- Stop hook → call a notification MCP server when a task completes
- PreToolUse hook → call a security scanning MCP tool before file writes
- FileChanged hook → call an MCP tool to sync changes to an external system

**Cross-system impact:**
- **Hooks (all projects):** The founder's current hooks (git-guard,
  auto-format, doc-drift-reminder) use `type: "command"`. The mcp_tool type
  opens richer automation without shell scripting.
- **CLAUDE.md template:** Document the five hook handler types (command, http,
  mcp_tool, prompt, agent) in the Hooks section.
- **Ops Pipeline:** MCP tool hooks could replace some Telegram relay patterns
  with direct MCP calls.
- **project-audit.md:** Add check for hook handler type diversity.

### 3. PostToolUse Hooks Include Tool Duration

CC v2.1.119 adds `duration_ms` to PostToolUse and PostToolUseFailure hook
inputs. This is the tool execution time excluding permission prompts and
PreToolUse hooks.

This enables performance-aware hook logic:
- Skip auto-formatting for sub-second edits (batch formatting at end)
- Alert on abnormally slow tool calls
- Build performance dashboards from hook telemetry
- Log tool execution patterns for workflow optimization

**Cross-system impact:**
- **auto-format hook:** Could conditionally skip Prettier for very fast edits
  to reduce overhead, or batch formatting via PostToolBatch instead.
- **Health System:** duration_ms data from hooks could feed into performance
  monitoring.
- **CLAUDE.md template:** Document duration_ms availability in hook inputs.

### 4. Auto Mode "$defaults" — Custom Rules Alongside Built-in Safeguards

CC v2.1.118 adds `"$defaults"` as a special token in `autoMode.allow`,
`autoMode.soft_deny`, and `autoMode.environment` settings. Including
`"$defaults"` adds custom rules alongside the built-in list instead of
replacing it entirely.

This means projects can customize auto mode behavior per-project while
retaining all built-in safety rules. Previously, setting custom auto mode
rules replaced the defaults entirely.

**Cross-system impact:**
- **CLAUDE.md template:** Can now include project-specific auto mode rules
  (e.g., always allow test commands, soft-deny database mutations) without
  losing the built-in safeguards.
- **Ops Pipeline:** Routines and scheduled tasks can have tailored auto mode
  rules per project context.

### 5. Context7 MCP Server — Version-Specific Library Docs in Prompts

Context7 by Upstash is an MCP server that fetches version-specific library
documentation directly into CC prompts, preventing hallucinated API
references. Free, open-source.

Install: `claude mcp add context7 -- npx -y @upstash/context7-mcp@latest`

When CC needs to use a library API, Context7 fetches the exact documentation
for the version in the project's package.json, not the training data version.
This avoids the common problem of CC using outdated or non-existent API
methods.

**Cross-system impact:**
- **All CC sessions:** Reduces hallucinated API calls, especially for
  fast-moving libraries (Next.js, Remotion, Playwright).
- **CLAUDE.md template:** Could recommend Context7 as a standard MCP server
  in the project setup phase.
- **new-project-master.md:** Add to Phase 2 (architecture) as an optional
  MCP server recommendation.

### 6. Codex Adversarial Review Plugin — Cross-Provider Code Review

The Codex Plugin for CC (`codex-plugin-cc` by OpenAI) runs adversarial code
review from GPT-4 in a separate process. The `/codex:adversarial-review`
command pressure-tests specific decisions (race conditions, caching,
security). Different training histories mean GPT-4 catches blind spots that
Claude misses.

Key: runs without consuming CC context window. Acts as a solo developer's
"second pair of eyes."

Requires GPT-4 API access (cost). Evaluate ROI for critical code paths only
(auth, payments, data mutations).

**Cross-system impact:**
- **Ops Pipeline:** Could integrate adversarial review into the pre-merge
  workflow alongside /ultrareview.
- **CLAUDE.md template:** If adopted, document as a review step for
  security-sensitive code.
- **Launch Checklist:** Could add adversarial review of auth/payment code
  as a pre-launch step.

## Worth Trying

### 7. Use /rewind to Undo Bad Responses — Avoids Context Pollution

From Marmelab's "Claude Code Tips I Wish I'd Had From Day One" (April 24):
when Claude produces a bad response, don't ask it to fix the mistake. The
wrong response stays in context and pollutes subsequent attempts. Instead,
use `/rewind` (or press Esc twice) to remove the bad turn entirely, giving
Claude a clean slate.

Additional pattern from the same article: use the preview/Chrome extension
not just for your own review, but to let Claude test its own changes and
self-correct errors before you report them.

**Cross-system impact:**
- **CLAUDE.md template:** Add to Learned Corrections: "Use /rewind for bad
  responses, don't ask Claude to fix its own mistakes."

## Bookmarked

### 8. DISABLE_UPDATES Env Var Pins CC Version

CC v2.1.118 adds `DISABLE_UPDATES` environment variable that completely
blocks all update paths including manual `claude update`. Stricter than
the existing `DISABLE_AUTOUPDATER`. Useful for pinning CC version in
routines, scheduled tasks, and CI pipelines where unexpected updates could
break behavior.

### 9. Remotion v4.0.448 — Clip-Path Transition Shapes

Remotion v4.0.448 (~April 20) adds clip-path support for transitions:
polygon, path, circle, ellipse, and inset shapes. Also includes Next.js
16.2.3 security fix and OpenAI Codex plugin support.

Minor but could enhance demo video transitions with more creative wipe
patterns. The demo pipeline works as-is; evaluate when next updating
video compositions.

### 10. human-cursor as ghost-cursor-playwright Replacement

ghost-cursor-playwright remains unmaintained (last release 2+ years ago).
CloverLabsAI has published `human-cursor` as an alternative specifically
designed for Playwright, addressing fundamental cross-compatibility issues
in the original ghost-cursor architecture.

Worth evaluating when next updating the demo recording pipeline. The current
ghost-cursor-playwright integration works but may accumulate compatibility
debt with newer Playwright versions.
